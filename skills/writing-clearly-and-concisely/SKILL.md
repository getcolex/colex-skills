# Writing Clearly and Concisely

**Announce at start:** "I'm using the writing-clearly-and-concisely skill to improve this writing."

William Strunk Jr.'s *The Elements of Style* (1918) teaches clear, concise writing.

**Core Principle**: ALL writing for humans is prose. Apply these rules to documentation, presentations, commit messages, error messages, UI text, and any explanation.

## When to Use This Skill

Use whenever writing prose for humans:
- Documentation, README files, technical explanations
- Presentations, pitch decks, business documents
- Error messages, UI copy, help text
- Reports, summaries, or any explanation
- Editing to improve clarity

**If a human will read it, apply these rules.**

## Elementary Principles of Composition

### Use Active Voice

| Bad (Passive) | Good (Active) |
|---------------|---------------|
| "The bug was fixed by the team" | "The team fixed the bug" |
| "Errors are handled by the system" | "The system handles errors" |
| "The proposal was reviewed by management" | "Management reviewed the proposal" |
| "This feature was built over 3 months" | "We built this feature over 3 months" |

### Put Statements in Positive Form

| Bad (Negative) | Good (Positive) |
|----------------|-----------------|
| "The feature is not very necessary" | "The feature is unnecessary" |
| "Did not remember" | "Forgot" |
| "He was not very often on time" | "He usually came late" |
| "does not have" | "lacks" |

### Omit Needless Words

| Bloated | Concise |
|---------|---------|
| "In my opinion, I think that..." | (just make the statement) |
| "at this point in time" | "now" |
| "allows users to be able to" | "lets users" |
| "the question as to whether" | "whether" |
| "the reason why is that" | "because" |
| "owing to the fact that" | "since" / "because" |
| "in spite of the fact that" | "although" |
| "call your attention to the fact" | "remind you" / "notify you" |
| "I was unaware of the fact that" | "I did not know" |
| "the fact that he had arrived" | "his arrival" |

### Use Definite, Specific, Concrete Language

| Vague | Specific |
|-------|----------|
| "The system processes data" | "The parser extracts JSON fields" |
| "handled appropriately" | "logged to stderr" |
| "A period of unfavorable weather set in" | "It rained every day for a week" |
| "Works with popular frameworks" | "Tested on Next.js, Remix, and Astro" |
| "Performance is significantly improved" | "Query time dropped from 847ms to 12ms" |

### Place Emphatic Words at End of Sentence

| Weak ending | Strong ending |
|-------------|---------------|
| "Humanity has hardly advanced in fortitude since that time, though it has advanced in many other ways" | "Humanity, since that time, has advanced in many other ways, but hardly in fortitude" |

### Keep Related Words Together

Subject and verb should not be separated by a phrase.

| Scattered | Together |
|-----------|----------|
| "Wordsworth, in the fifth book of The Excursion, gives a minute description of this church" | "In the fifth book of The Excursion, Wordsworth gives a minute description of this church" |

## Business Writing Anti-Patterns

### Avoid Corporate Fluff

| Fluff | Direct |
|-------|--------|
| "We are excited to announce..." | (just announce it) |
| "We're thrilled to share..." | (just share it) |
| "empowering developers to leverage synergies" | "helps developers work faster" |
| "best-in-class solutions" | (describe what it actually does) |
| "seamlessly integrate" | "works with" |
| "drive innovation through collaborative paradigms" | (delete this entirely) |

### Remove Hedging

| Hedging | Confident |
|---------|-----------|
| "This might help you potentially improve..." | "This cuts time in half" |
| "You could possibly consider..." | "Try this" |
| "This may or may not be useful..." | (just say what it does) |
| "Some users have reported that..." | "Three users found X" |

### Cut Filler Phrases

| Filler | Better |
|--------|--------|
| "Let's dive deep into..." | (just start) |
| "Without further ado..." | (delete) |
| "At the end of the day..." | (delete) |
| "It goes without saying..." | (if true, don't say it) |
| "In conclusion..." | (your final point should be obvious) |

## Common Rationalizations to Reject

| Excuse | Reality |
|--------|---------|
| "Write quickly → skip editing" | Clarity ALWAYS worth the time |
| "Just internal docs" | Internal docs need clarity MORE, not less |
| "Technical writing ≠ prose" | ALL writing for humans is prose |
| "Not formal prose" | Formality ≠ clarity; informal needs clarity too |
| "Clarity isn't critical here" | This is NEVER true. Clarity is always critical |
| "Good enough for now" | Poor writing wastes readers' time forever |

## Application Process

When reviewing or writing prose:

1. **Read through once** to understand the content
2. **Apply active voice** - flip every passive construction
3. **Cut needless words** - remove every word that adds nothing
4. **Make specific** - replace vague language with concrete details
5. **Check endings** - ensure emphatic words land at sentence ends
6. **Remove fluff** - delete corporate speak and hedging
7. **Read aloud** - if it sounds stuffy, rewrite

## Bottom Line

Writing for humans? Apply these rules. Under pressure? Apply rules anyway. Clarity is always worth 30 seconds.

**Don't accept rationalizations. All writing for humans deserves clarity.**
